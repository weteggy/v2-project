What We Proved

⚡ Framework-Agnostic Generation

  • Same spec.json produces correct components across 4 frameworks
  • Angular: standalone components, signal inputs, OnPush
  • React: functional components, TypeScript, CVA variants
  • Vue: Composition API, script setup, defineProps
  • Web Components: Lit, light DOM, custom element registration
  • No parallel codebases — CLAUDE.md handles framework patterns

🎨 Multi-Product Theming

  • Components use semantic tokens, never raw colors
  • Swap a CSS theme file — components re-skin automatically
  • MEFF to Discover swap validated with zero component changes
  • OKLCH algorithm generates accessible palettes from seed values

📋 Spec-as-Source-of-Truth

  • 5 structured spec.json contracts define component APIs
  • Wrong output? Update the spec, regenerate — don't hand-edit
  • Specs include props, states, a11y rules, usage guidelines
  • Machine-readable format any AI tool can consume

🤖 AI as Infrastructure

  • Claude Code generates correct UI from specs + CLAUDE.md alone
  • Design system becomes an API for LLMs
  • Positions Athena as the structured data layer for AI-first products
  • Works today — not a future aspiration

Data Visualization

Data should feel native to the system — not bolted on.

We're expanding our existing data visualization color guidance into full theming support for ag-Grid tables and Highcharts — so every chart, table, and data surface looks, feels, and behaves as part of the design system. Same tokens. Same semantic colors. Same brand consistency.

What We're Building

🎨

Color System

Extending the OKLCH token palette with data-specific sequences — categorical palettes, sequential scales, diverging gradients, and semantic status colors tuned for charts and grids.

Existing guidance Expanding
📊

Highcharts Theme

A complete Highcharts theme that maps to our semantic tokens — chart backgrounds, axis styling, tooltip design, series colors, legend treatment, and responsive breakpoints. Every chart type gets the system's look for free.

New Phase 2
🗃

ag-Grid Theme

Custom ag-Grid theme using our design tokens — row styling, header treatment, cell typography, selection states, sorting/filtering indicators, and conditional formatting that matches the system's status colors.

Why This Matters

Data is the product

  • NIQ products are data-heavy — grids and charts are the core experience
  • If data surfaces don't match the system, users notice the inconsistency first
  • Themed ag-Grid and Highcharts close the biggest visual gap

One token change, everywhere

  • Brand refresh? Update seed colors once — buttons, grids, and charts update together
  • New product theme? One theme file covers UI + data viz + tables
  • Dark mode? Token layer handles all three surfaces in one pass
